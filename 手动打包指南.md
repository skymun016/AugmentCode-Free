# Python 3.13+ 手动打包指南

由于Python 3.13引入了外部管理环境限制，如果自动脚本失败，可以按以下步骤手动打包：

## 方法一：使用虚拟环境（推荐）

```bash
# 1. 创建虚拟环境
python3 -m venv build_env

# 2. 激活虚拟环境
source build_env/bin/activate  # macOS/Linux
# 或 build_env\Scripts\activate  # Windows

# 3. 升级pip
pip install --upgrade pip

# 4. 安装依赖
pip install -r build_requirements.txt

# 5. 运行打包
python build.py

# 6. 退出虚拟环境
deactivate

# 7. 清理虚拟环境（可选）
rm -rf build_env
```

## 方法二：使用系统包模式

```bash
# 直接使用 --break-system-packages 参数
pip install --break-system-packages -r build_requirements.txt
python build.py
```

## 方法三：使用用户模式

```bash
# 安装到用户目录
pip install --user -r build_requirements.txt
python build.py
```

## 方法四：使用pipx（如果可用）

```bash
# 安装pipx
brew install pipx  # macOS

# 使用pipx运行
pipx run --spec pyinstaller pyinstaller --onefile main.py
```

## 自动脚本使用

更新后的脚本会自动检测Python版本：
- Python 3.12+：自动使用虚拟环境方式
- Python < 3.12：使用标准方式

```bash
./build.sh  # macOS/Linux
build.bat   # Windows
```

## 如果仍然失败

请手动执行以下命令：

```bash
# 创建虚拟环境并打包
python3 build_with_venv.py
```

这个脚本会自动创建虚拟环境、安装依赖、打包，然后清理环境。
